# A2 Work-item Detail + Agent Monitor — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the work-item Detail screen (Screen D — breadcrumb, item header, tabs, Spec body + right rail) and the Agent Monitor tab (Screen D2 — step timeline, log stream, token meter), wired to the mock data layer.

**Architecture:** `DetailScreen` (replacing the placeholder) loads the item via `useWorkItem(itemId)` and renders the breadcrumb, `ItemHeader`, a `TabBar` (Spec · Attachments · Activity · Agent · Subagents), and the active tab body. The Spec tab renders the item's markdown `spec`; a right rail shows Properties/Token-usage/Activity/Attachments. The Agent tab renders `AgentMonitor`, which uses `useRun` (snapshot + SSE) for the step timeline and log stream.

**Tech Stack:** Plan-1 `components/ui`, Plan-2 `lib/api` (`useWorkItem`, `useRun`), react-router params, `react-markdown` (new dep) for the spec body.

## Global Constraints

- pnpm; commands from `projects/ui/`. Compose existing primitives + hooks. Exact values for Screen D / D2 are in `docs/design/README.md` § "Screen D — Work Item Detail (Spec tab, default)" and "Screen D2 — Work Item Detail (Agent Monitor tab)". Cite the subsection.
- TypeScript strict; props typed; no `any`; types from `components["schemas"]` (`WorkItem`, `AgentRun`, `RunStep`, `LogLine`). Tokens via Tailwind utilities. Immutable. Commit `<type>: <description>`; one per task. TDD. Keep `pnpm test`/`pnpm lint`/`pnpm build` green each task.
- Tests use a real seeded work item: read `src/lib/api/mocks/fixtures` / `db.ts` for a seeded work-item id (and a run id for the monitor). Route the Detail tests through `createMemoryRouter(routes, {initialEntries:["/projects/<pid>/items/<id>"]})`.
- Work in the `feat/a2-ui` worktree; Plans 1–4 are on this branch.
