# A2-4 Live-API Wiring Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the A1 backend API emit/accept the UI's OpenAPI contract shapes for Projects, WorkItems, and Teams/AgentDefinitions, then run the UI hybrid (live for those three groups, MSW-mocked for the rest).

**Architecture:** A genuine domain change (work-item status → the UI's 5-set) plus additive fields (`priority`, `enabled`, `tokenLimit`); all other translation (camelCase, `type`/`spec`/`model` renames, computed `itemCount`/`epicId`, `null` agent-run fields) lives in the **API interactor layer** as contract schemas + domain↔contract mappers, with the generic `CrudRouter` extended to take mapper hooks. The UI needs no data mapping — only an MSW split behind a `VITE_LIVE_API` flag so the live paths bypass to the real `/api`.

**Tech Stack:** Backend = Python 3.12 / FastAPI / Pydantic v2 / SQLAlchemy 2.0 / Alembic / pytest. UI = Vite / React / TS / MSW / vitest.

## Global Constraints

- Backend: `uv`; `make test` / `make coverage` (80% gate) / `make lint` (ruff + mypy) stay green. Domain stays pure (no I/O, no adapter imports); contract adaptation lives in `interactors/api/`.
- UI: `pnpm` (never npm); `pnpm test` / `pnpm lint` / `pnpm build` stay green. The UI's generated types are the contract — do NOT hand-edit `src/lib/api/schema.d.ts`; the backend conforms to it.
- **Status set is the UI-canonical 5:** `backlog`, `todo`, `in_progress`, `in_review`, `done`. **Migration remap:** `to_do→todo`, `approved→done`, `blocked→todo`, `failed→todo`.
- Envelope `{success,data,error}` (+`meta`) on every response — unchanged. Owner-scoping unchanged; `owner_id` is never surfaced in the contract. Immutability via `model_copy`. UUID-hex ids. TDD; AAA tests; `<type>: <description>` commits, one per task.
- **The contract is authoritative.** Exact UI field names/types come from `projects/ui/src/lib/api/schema.d.ts` (generated) and `projects/ui/openapi/naaf-api.yaml`. Each backend mapper must match those exactly — read them before writing a mapper.
- Work in the `feat/a2-4-live-api` worktree at `.worktrees/a2-4-live-api`.

from domain.errors import InvalidTransition
from domain.work_item import WorkItemStatus as S

# Directed edges of the work-item lifecycle (UI-canonical 5-state set).
ALLOWED_TRANSITIONS: dict[S, set[S]] = {
    S.BACKLOG: {S.TODO},
    S.TODO: {S.BACKLOG, S.IN_PROGRESS},
    S.IN_PROGRESS: {S.IN_REVIEW, S.TODO},
    S.IN_REVIEW: {S.IN_PROGRESS, S.DONE},
    S.DONE: set(),
}


def validate_transition(current: S, target: S) -> S:
    """Return target if the current->target edge is legal, else raise."""
    if target not in ALLOWED_TRANSITIONS.get(current, set()):
        raise InvalidTransition(
            f"Cannot transition from {current.value} to {target.value}"
        )
    return target
```

def _create(self) -> Callable:
    def create_record(obj_in: self.create_schema, uow=Depends(self.db_dependency)):  # type: ignore[name-defined]  # noqa: B008
        domain_in = self.to_domain_create(obj_in) if self.to_domain_create else obj_in
        return ok(self._respond(self._repo(uow).create(domain_in)))
    return create_record

def _read(self) -> Callable:
    def read_record(id: UUID, uow=Depends(self.db_dependency)):  # noqa: B008
        return ok(self._respond(self._repo(uow).read(id.hex)))
    return read_record

def _read_multi(self) -> Callable:
    def read_multiple(uow=Depends(self.db_dependency), filters: str = "{}",  # noqa: B008
                      page_size: int = 50, page_number: int = 1, order_by: str = "-created_at"):
        page = self._repo(uow).read_multi(filters=json.loads(filters), page_size=page_size,
                                          page_number=page_number, order_by=order_by)
        return ok([self._respond(r) for r in page.results],
                  meta={"total": page.total, "page_size": page.page_size, "page_number": page.page_number})
    return read_multiple

def _update(self) -> Callable:
    def update_record(id: UUID, obj_in: self.update_schema, uow=Depends(self.db_dependency)):  # type: ignore[name-defined]  # noqa: B008
        domain_in = self.to_domain_update(obj_in) if self.to_domain_update else obj_in
        return ok(self._respond(self._repo(uow).update(id.hex, domain_in)))
    return update_record
```
