# A2 Inbox — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the Inbox screen (Screen G) — a notification list pane (356px, filter tabs + badges) beside a conversation pane (agent/user bubbles + quick actions + reply input), wired to the mock data layer.

**Architecture:** `InboxScreen` (replacing the placeholder) loads notifications via `useInbox(filter?)` and renders `InboxList` (filterable, selectable) + `ConversationPane` for the selected item, loading its conversation via a `useInboxConversation(id)` hook. Selection is route-driven (`/inbox/:id`) with a sensible default.

**Tech Stack:** Plan-1 `components/ui` (`StatusBadge`, `Avatar`, `Button`), Plan-2 `lib/api` (`useInbox`, `useThreads`/messages), react-router.

## Global Constraints

- pnpm; commands from `projects/ui/`. Compose existing primitives + hooks. Exact values in `docs/design/README.md` § "Screen G — Inbox" (notification list pane, notification item, badge types, extended conversation pane). Cite the subsection.
- TypeScript strict; props typed; no `any`; types from `components["schemas"]` (`InboxItem`, `Message`). Tokens via Tailwind utilities. Immutable. Commit `<type>: <description>`; one per task. TDD. Keep `pnpm test`/`pnpm lint`/`pnpm build` green each task.
- Inbox item `type` is `action_needed | review_needed | info | resolved` → the `StatusBadge` `BadgeKind` (Plan 1) renders the badge. The list filter tabs are All · Action needed · Info · Resolved.
- Tests read seeded inbox items from the mock fixtures (`seed.inboxItems`); confirm the export and adapt if needed. Route inbox tests through `createMemoryRouter(routes, {initialEntries:["/inbox"]})`.
- Work in the `feat/a2-ui` worktree; Plans 1–5 are on this branch.

**Interfaces:**
- Consumes: `InboxItem` type; Plan-1 `StatusBadge`, `Avatar`; `useInbox`.

  - `NotificationItem({ item, selected, onSelect }: { item: InboxItem; selected: boolean; onSelect: (id: string) => void })` — a 13px-padded row per `docs/design/README.md` § Inbox "Notification item": `StatusBadge` (mapped from `item.type`), timestamp, title (bolder when unread), preview, agent/item footer; selected → accent left-border; resolved → reduced opacity. Maps `item.type` → `BadgeKind` (the values match: `action_needed/review_needed/info/resolved`).
  - `InboxList({ selectedId, onSelect }: { selectedId?: string; onSelect: (id: string) => void })` — the 356px pane per § Inbox: header (title + unread count + "Mark all read"), filter tabs (All · Action needed · Info · Resolved → maps to `useInbox(filter)`), and the `NotificationItem` rows. Local `useState` for the active filter tab.
